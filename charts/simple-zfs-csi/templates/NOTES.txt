{{- if eq .Values.toolbox.kind "DaemonSet" }}
   One toolbox runs per storage node. ZFS is node-local, so exec into the pod on
   the node that owns the target pool:

     NODE=$(kubectl get zfspool <guid> -o jsonpath='{.status.currentNode}')
     POD=$(kubectl -n {{ .Release.Namespace }} get pod -l app.kubernetes.io/component=toolbox \
       --field-selector spec.nodeName=$NODE -o jsonpath='{.items[0].metadata.name}')
     kubectl -n {{ .Release.Namespace }} exec -it $POD -- zpool status

NOTE ON UPGRADES: the NetworkExport and ZfsPool CRDs are installed from the chart's
crds/ directory. `helm upgrade` does NOT update them (no Helm flag does). After
upgrading to a release that changes a CRD schema, apply the CRDs manually first:

     kubectl apply -f charts/simple-zfs-csi/crds/     # local checkout; or `make install-crd`
     # published chart: helm pull ... --untar && kubectl apply -f simple-zfs-csi/crds/
